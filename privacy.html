개인정보처리방침

시행일: 2026년 3월 9일 | 버전 1.1

PEPick!(이하 "서비스")은 「개인정보 보호법」 제30조에 따라 이용자의 개인정보를 보호하고, 관련 고충을 신속하게 처리하기 위하여 다음과 같이 개인정보처리방침을 수립·공개합니다. 본 서비스는 초등학교 체육 수업을 지원하는 교육용 웹 애플리케이션으로, 교사(이용자)와 교사가 입력하는 학생 정보를 처리합니다.

제1조 (수집하는 개인정보 항목)

1. 교사(이용자) 정보 — Google OAuth 로그인

수집 항목 수집 목적 법적 근거
Google 계정 이름 (displayName) 서비스 내 교사 이름 표시 정보주체의 동의
Google 이메일 주소 계정 식별, 로그인 인증 정보주체의 동의
Google 프로필 사진 URL 서비스 내 프로필 이미지 표시 정보주체의 동의

서비스는 이용자의 Google 비밀번호를 저장하지 않으며, Google OAuth 인증 토큰을 통해서만 인증합니다.

2. 학생 정보 — 교사 직접 입력

수집 항목 수집 목적 법적 근거
학생 이름 술래뽑기·모둠뽑기 시 학생 식별 교사의 교육 목적 수행을 위한 동의 하 입력
학생 번호 학급 내 학생 구분 교사의 교육 목적 수행을 위한 동의 하 입력
학생 성별 성별 기반 모둠 구성 교사의 교육 목적 수행을 위한 동의 하 입력

학생 정보는 교사가 직접 입력하며, 학생이 서비스에 직접 가입하거나 로그인하지 않습니다. 학생의 민감 정보(주민등록번호, 건강 정보 등)는 수집하지 않습니다.

3. 자동 수집 정보

수집 항목 수집 목적 법적 근거
기기 정보 (User-Agent, 화면 해상도) 서비스 최적화, 호환성 확보 정당한 이익 (서비스 개선)
접속 로그 (IP 주소, 접속 시간) 보안, 부정 이용 방지 정당한 이익 (보안)

제2조 (개인정보의 수집 및 이용 목적)

서비스는 수집한 개인정보를 다음의 목적으로만 이용합니다.

  1. 서비스 계정 관리 및 본인 확인 — Google OAuth를 통한 로그인·인증
  2. 체육 수업 보조 기능 제공 — 술래뽑기, 모둠뽑기, 배지 관리, 학급 통계 등 교육 목적 기능 제공
  3. 서비스 개선 및 이용 통계 분석 — 기능 사용 패턴 분석을 통한 서비스 품질 향상
  4. 불법·부정 이용 방지 및 서비스 안정성 확보 — 비정상 접근 탐지 및 차단

제3조 (학생 데이터 처리 및 교사의 책임)

교사가 서비스에 직접 입력하는 학생 정보(이름, 번호, 성별 등)는 교사 본인의 교육 활동 목적으로만 처리됩니다.

  • 학생 데이터는 교사의 Firebase 계정 하위에 격리 저장됩니다.
  • 다른 교사 또는 제3자가 접근할 수 없습니다.
  • 서비스 운영자는 교사의 동의 없이 학생 데이터를 열람·분석하지 않습니다.

교사의 의무

이용자(교사)는 학생 데이터를 입력·관리함에 있어 다음의 의무를 준수해야 합니다.

  1. 고지 및 동의: 학생 및 학부모(법정대리인)에게 본 서비스의 사용 사실을 사전에 고지하고, 필요한 경우 동의를 받아야 합니다.
  2. 최소 수집: 교육 목적 달성에 필요한 최소한의 정보만 입력해야 합니다.
  3. 적시 삭제: 학기 종료, 전학, 졸업 등으로 더 이상 필요하지 않은 학생 데이터는 지체 없이 삭제해야 합니다.
  4. 화면 노출 주의: 수업 중 프로젝터·TV 등으로 화면을 공개할 때 다른 학생의 개인정보가 불필요하게 노출되지 않도록 주의해야 합니다.

면책

교사가 위 의무를 위반하여 발생하는 개인정보 침해, 분쟁, 법적 책임은 해당 교사에게 있으며, 서비스 제공자는 교사의 의무 위반으로 인한 결과에 대해 책임을 지지 않습니다.

제4조 (14세 미만 아동정보 보호)

본 서비스는 초등학생(14세 미만 아동)의 데이터를 다루는 교육용 서비스로서, 「개인정보 보호법」 제22조 및 관련 규정에 따라 다음과 같이 아동정보를 보호합니다.

  1. 직접 가입 불허: 14세 미만 아동(학생)은 본 서비스에 직접 가입하거나 로그인할 수 없습니다. 학생 정보는 오직 교사를 통해서만 간접적으로 관리됩니다.
  2. 교사를 통한 간접 관리: 교사가 교육 목적으로 학생의 이름, 번호, 성별만을 입력하며, 학생에게 계정이 부여되지 않습니다.
  3. 법정대리인의 권리: 학부모(법정대리인)는 교사 또는 서비스 제공자에게 자녀의 개인정보에 대한 열람, 수정, 삭제를 요청할 수 있습니다.
  4. 민감 정보 미수집: 학생의 주민등록번호, 생년월일, 주소, 건강 정보 등 민감한 개인정보는 일체 수집하지 않습니다.

구분 보유 기간 파기 시점
교사(이용자) 정보 서비스 이용 기간 회원 탈퇴 시 즉시 삭제
학생 정보 교사 관리 하 보유 학급 삭제 또는 교사 탈퇴 시 삭제
자동 수집 정보 수집일로부터 1년 보유 기간 만료 시 자동 삭제

  • 관련 법령에 의해 보존 의무가 있는 경우, 해당 기간 동안 보관합니다.

제6조 (개인정보의 처리 위탁)

서비스는 원활한 운영을 위해 다음과 같이 개인정보 처리를 위탁하고 있습니다.

수탁업체 서버 위치 위탁 업무 개인정보 보호 정책
Firebase (Google Cloud) asia-northeast3 (서울) 데이터 저장 (Firestore), 사용자 인증 (Auth) Google 개인정보처리방침
Netlify 미국 (CDN 글로벌) 웹사이트 호스팅, 서버리스 함수 실행 Netlify Privacy Policy

서비스는 위탁 계약 시 개인정보가 안전하게 관리될 수 있도록 필요한 사항을 규정하고 있으며, 수탁업체가 개인정보를 안전하게 처리하는지 감독합니다.

제7조 (개인정보의 파기 절차 및 방법)

서비스는 개인정보의 보유 기간이 경과하거나 처리 목적이 달성된 경우 지체 없이 해당 정보를 파기합니다. 이용자는 다음의 방법으로 직접 데이터를 삭제할 수 있습니다.

삭제 대상 앱 내 경로 기술적 처리
회원 탈퇴 (전체 삭제) 설정 > 계정 관리 > 탈퇴하기 (2단계 확인) Firestore 사용자 문서 및 하위 데이터 전체 삭제 → localStorage 전체 삭제 → Firebase Auth 계정 삭제
학급 삭제 메인 화면 > 학급 카드 > 삭제 버튼 해당 학급의 학생 서브컬렉션 삭제 → 학급 문서 삭제
로컬 데이터 초기화 설정 > 계정 관리 > 데이터 초기화 브라우저 localStorage의 pet_* 키 전체 삭제 (Firestore 데이터는 유지)

전자적 파일은 복구·재생이 불가능한 방법으로 영구 삭제하며, 종이 문서는 해당 사항이 없습니다.

제8조 (이용자의 권리와 행사 방법)

이용자는 언제든지 다음의 권리를 행사할 수 있습니다.

  • 개인정보 열람 요청
  • 개인정보 수정·삭제 요청
  • 개인정보 처리 정지 요청
  • 회원 탈퇴 및 데이터 삭제 요청

권리 행사는 아래 개인정보보호 책임자에게 이메일로 요청할 수 있으며, 요청일로부터 10일 이내에 조치하겠습니다. 권리 행사는 이용자의 법정대리인이나 위임을 받은 자를 통해서도 할 수 있습니다.

제9조 (자동 수집 정보 및 쿠키·로컬 저장소 정책)

1. 쿠키 (Cookie)

서비스는 별도의 쿠키를 사용하지 않습니다. 인증은 Firebase SDK의 자체 메커니즘을 통해 처리됩니다.

2. 로컬 저장소 (localStorage)

서비스는 오프라인 사용과 빠른 응답을 위해 브라우저의 localStorage에 다음 데이터를 저장합니다. 이 데이터는 서버로 전송되지 않으며, 이용자의 기기에만 저장됩니다.

키 저장 내용 삭제 방법
pet_classes 학급 목록 및 학생 데이터 (로컬 캐시) 데이터 초기화 또는 학급 삭제
pet_tag_game 술래뽑기 설정 및 이력 데이터 초기화
pet_group_manager 모둠뽑기 설정 및 이력 데이터 초기화
pet_badges 배지 데이터 데이터 초기화
pet_settings 사용자 설정 (기본 모둠이름 등) 데이터 초기화
pet_teacher 교사 프로필 캐시 데이터 초기화 또는 회원 탈퇴
pet_consent 약관 동의 상태 (버전, 동의 일시) 회원 탈퇴

3. Service Worker

서비스는 오프라인 캐싱을 위해 Service Worker를 사용합니다. Service Worker는 앱 셸(HTML, CSS, JS)만 캐싱하며, 개인정보를 포함하지 않습니다.

제10조 (가명정보의 처리)

서비스는 현재 가명정보를 처리하지 않습니다. 향후 통계·연구 목적으로 가명처리를 시행하게 되는 경우, 「개인정보 보호법」 제28조의2에 따라 별도로 공지하겠습니다.

제11조 (개인정보 보호를 위한 기술적·관리적 대책)

서비스는 개인정보의 안전한 처리를 위해 다음과 같은 조치를 취하고 있습니다.

  • 접근 통제: Firebase Security Rules를 통한 사용자별 데이터 격리 (request.auth.uid == userId)
  • 전송 암호화: HTTPS(TLS) 통신 암호화
  • 웹 보안: XSS, CSRF 등 웹 보안 취약점 대응
  • 보안 헤더: Content-Security-Policy, X-Frame-Options, X-Content-Type-Options 등 적용
  • 최소 권한 원칙: Google OAuth 로그인 시 최소한의 프로필 정보만 요청

제12조 (개인정보보호 책임자)

서비스는 개인정보 처리에 관한 업무를 총괄해서 책임지고, 이용자의 개인정보 관련 불만 처리 및 피해 구제 등을 위하여 아래와 같이 개인정보보호 책임자를 지정하고 있습니다.

제14조 (교육 관련 법령 준수)

본 서비스는 교육 목적의 보조 도구로서, 이용자(교사)는 서비스 이용 시 다음 법령 및 규정을 준수해야 합니다.

  • 「초·중등교육법」 및 동법 시행령: 학생 개인정보의 수집·이용·제공에 관한 규정
  • 「교육기본법」: 학습자의 기본적 인권 존중에 관한 규정
  • 학교 내부규정: 각 학교의 개인정보보호 지침 및 정보화 관련 규정

교사는 소속 학교의 내부 규정에 따라 외부 디지털 도구의 사용에 대한 승인이 필요한 경우, 사전에 관련 절차를 이행해야 합니다.

  2. 방침이 변경되는 경우 변경 사항은 서비스 내 공지 또는 동의 절차를 통해 안내합니다.
  3. 이용자에게 불리한 변경이 있는 경우, 시행일 30일 전에 사전 공지합니다.
